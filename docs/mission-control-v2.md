# Mission Control Current Contract

This file is retained at its historical path because many agent notes still point here. As of May 2, 2026, it describes the current Mission Control contract.

## Canonical Meaning

An unqualified reference to "Mission Control" means the current Josh 2.0 React kiosk:

- Source: `v2-react/`
- Build output: `dist/v2-react/`
- Live Josh 2.0 URL: `http://127.0.0.1:5174/`
- Kiosk launcher: `scripts/open_mission_control_kiosk.sh`
- Build command: `npm run build`
- Dev command: `npm run dev`

Agents should not call this surface "v1", "v2", or "React v2" in new operator-facing notes. Use "Mission Control" or "current React Mission Control" unless the subject is explicitly historical migration or rollback.

## Legacy Surfaces

The older surfaces remain available only for compatibility and rollback:

- `index.html`: legacy static dashboard.
- `v2/index.html`: older static proof surface for the initial Supabase state migration.
- `scripts/open_react_v2_kiosk.sh`: compatibility alias that now delegates to `scripts/open_mission_control_kiosk.sh`.

Do not point Josh 2.0, JAIMES, J.AI.N, OpenCLAW, Hermes, or Codex handoffs at the legacy static pages for normal Mission Control work.

## Data Contract

The current React Mission Control reads from:

- Supabase dashboard-safe rows when configured.
- Local sidecars under `data/` as fallback.
- Model usage and signal sidecars generated by `scripts/update_mission_control.py`.

For Codex/JOSHeX work, publish visible status with:

```bash
python3 scripts/agent_publish.py --agent joshex --brain-feed --type status --status active --title "..." --detail "..."
```

For jobs, add `--job` when the work should appear in Today's Jobs.

## Verification Contract

Before calling current Mission Control UI work complete:

1. Build from `mission-control/`:

```bash
npm run build
```

2. Sync changed React source to Josh 2.0 if working from another machine.
3. Reload/verify the Josh 2.0 kiosk at `http://127.0.0.1:5174/`.
4. Confirm the visible kiosk has no overflow and shows the current Brain Feed layout.
5. Publish final Brain Feed status with `--agent joshex --brain-feed`.

## Historical Note

The original v2 migration introduced Supabase tables and dashboard-safe publishers while the static dashboard remained the primary operator surface. That migration state is no longer the default mental model. The Supabase-backed data layer remains useful, but the current operator surface is the React kiosk described above.
